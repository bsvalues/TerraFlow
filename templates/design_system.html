Body Text

Lead paragraph is used for introductory text. It's slightly larger and has a lighter font weight to stand out.

Font: var(--font-family-base, "Inter") | Size: 1.25rem | Weight: 300

Regular paragraph text is used for most body content. It should have good readability and be appropriately sized for reading long-form content.

Font: var(--font-family-base, "Inter") | Size: var(--font-size-body, 16px) | Weight: var(--weight-regular, 400)

Small text is used for less important information, captions, footnotes, etc.

Font: var(--font-family-base, "Inter") | Size: 0.875rem | Weight: var(--weight-regular, 400)

Loading Indicators

The TerraFusion platform uses standardized loading indicators to provide consistent feedback for all data operations.

Loadable Content

This container can display a loading indicator overlay when data is being fetched or processed.

// Show loading indicator on a container
const loadingId = terraFusionLoading.show('#myContainer', {
  message: 'Loading data...',
  spinnerColor: 'primary',
  spinnerSize: 'lg'
});

// Hide loading indicator when done
terraFusionLoading.hide(loadingId);

Accessibility Guidelines

Accessibility First

The TerraFusion Platform is designed with accessibility in mind. All components should meet WCAG 2.1 AA standards.

Color Contrast

All text should have sufficient contrast with its background to ensure readability. The minimum contrast ratios are:

  • Normal text: 4.5:1
  • Large text (18pt or 14pt bold): 3:1
  • UI components and graphics: 3:1
Good Example: 7:1 ratio

This text has good contrast with the background, making it easy to read for most users.

Poor Example: 2:1 ratio

This text has poor contrast with the background, making it difficult to read for many users.

Keyboard Navigation

All interactive elements should be accessible via keyboard navigation. This includes:

  • Using Tab to navigate between elements
  • Using Space or Enter to activate buttons
  • Using arrow keys to navigate within components like selects and date pickers
  • Providing visible focus indicators

Screen Reader Support

All content should be accessible to screen readers. This includes:

  • Using semantic HTML elements
  • Providing alt text for images
  • Using ARIA attributes where necessary
  • Ensuring dynamic content changes are announced
